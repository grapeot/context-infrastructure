# K05 From Users to Builders

## 核心表述

AI 时代，技能（出卖时间换报酬）在结构性贬值，数字资产（可复利增长的产品/内容/代码）在结构性升值。每个人都应逐步发展建造能力，但不是立即辞职创业——在打工的"动物园"里保持"丛林捕猎"能力。Builder 本身是阶梯中间级，Architect（系统构建者）是终态。

## 展开

这是课代表给受众最核心的身份召唤。他用"赛博佃农"来形容纯粹的平台使用者——你在别人的平台上创造内容、提供服务，平台抽成或改规则你就失去一切。Builder 的核心区别是你拥有自己的数字资产：代码产品、自媒体账号（storefront）、社区、作品集。

AI 降低了 Builder 的门槛——代码变便宜了（K07 AI-native），但 PMF 的门槛没变。他明确说"你想让这个产品真的能解决大家的问题，大家真的愿意去使用它，这是一个难题"。所以 Builder 不是"学会用 Cursor 写个贪吃蛇"，而是"能交付解决真实问题的产品"。

从 2025 年底开始，他的观点进一步演化：Builder 红利正在消退，真正稀缺的是 Architect——能设计系统、理解架构、在 AI agent 时代编排多个 agent 协作的人。

## 边界条件

- **不是"辞职创业"**：他用"动物园里的老虎"比喻，建议保持打工同时发展建造能力。建产品是 5 年计划，不是明天的事。
- **Builder 是中间态**：2025+ 他认为纯 Builder 红利在消退，Architect 是下一个阶梯。
- **PMF 仍然很难**：代码变便宜不等于产品变容易。执行层（Craft）的壁垒依然存在。
- **与 C08 互补**：Builder 是 how（建造能力路径），定义自己的游戏（K06）是 why（逃离空洞的外部规则）。

## 代表性证据

1. "你在这里一定要做一个 builder，你才有可能抓到这个机会。User 是抓不到的。你整天跟那个 ChatGPT 对话，我觉得这个没有用。得要做得更深一层到两层。" [YouTube: 起跑线重置, 2024-09-24]

2. "哪怕你在动物园里做老虎，你也应该保持丛林里边捕猎的能力。" [YouTube: 最大化赚钱能力的具体路径, 2025-04-02]

3. "有了这个元能力，都可以自学，而且在我们看来，掌握自学的习惯和信心，比喂知识，重要的多。" [Circle: 当知识获取的门槛降为零时, 2026-01-05]

4. "技能是在贬值的......资产是在升值的......要把技能转化为资产。" [YouTube: 技能转换为资产, 2026-01-22]

## 跨源表现

- **YouTube**：最完整的论述。从"User→Builder"到"Builder→Architect"的演化在视频中可追溯。
- **Circle**：应用到课程设计——课程不是教知识，是培养 Builder 能力。
- **微信**：社区成员内化这一概念（"任何伟大的成就都源于创造"）。

## 可信度

8.5/10。跨源一致、时间稳定、有清晰的演化轨迹。减分原因：Architect 层的论述还在形成中（2025 年底才开始），尚未充分展开。
