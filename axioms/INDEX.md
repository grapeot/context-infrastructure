## 预测场景速查

每条 axiom 什么时候用、什么时候不用。

| 编号 | 触发场景（用时） | 失效场景（不用时） |
|------|-----------------|-------------------|
| K01 | 面对新领域/新决策时，预测他会先问"底层因果是什么" | 纯执行层的操作问题（他不会把拧螺丝上升到世界模型） |
| K02 | 知识层面的分歧/争论 | 对方认知基础不足时他跳过 steelman 直接评判；利益相关的运营决策有延迟 |
| K03 | 选赛道、评估是否坚持、短期 vs 长期权衡 | 冷启动阶段（此时快速验证 > 耐心积累）；catastrophic risk 场景 |
| K04 | 分析复杂现象时（AI 评估、行业判断、人物评价） | 需要快速决策时他也做单维判断（决策模式 vs 分析模式） |
| K05 | 职业方向建议、AI 学习路径建议 | 不是说立刻辞职——"动物园里保持捕猎能力" |
| K06 | 有人抱怨现状但不行动时 | 职业早期需要在别人的框架里学习时 |
| K07 | 有人问"该先想还是先做" | 方向已经清楚只需要执行时，直接用执行层（Craft+行动） |
| K08 | 技术 builder 问"为什么产品没人用" | 产品质量本身还没达标时（产品仍是必要条件） |
| K09 | AI 编程方法论讨论 | 纯 vibe coder 没有 architect 能力时 |
| K10 | 社区治理决策 | 规模极大需要分权治理时 |
| K11 | 有人抱怨环境不公平时 | 短期生存压力下必须接受次优环境时 |
| K12 | 评估年轻人潜力、建立 mentor 关系时 | 纯一次性交易 |
| K13 | 定价、pitch、landing page 决策（私下） | 公开场合他不会说"定价是讲故事" |
| K14 | 理解他为什么高强度工作不 burnout | 不喜欢的事他也偷懒（锻炼、行政） |
| KS01 | 模拟他的内容结构 | 他的 taxonomy/lecture 模式约 40% 框架先行 |
| KS02 | 模拟他的独白/反思内容 | 纯教程和嘉宾主导访谈中不出现 |
| KS03 | 模拟他的 Circle 长文 | 教程类内容 |
| KS04 | 模拟他的视频标题和开场 | ~40% 的视频不用反直觉 hook |
| KS05 | 预测他如何处理公共攻击 | 私下对不达标的人他直接评判不做分流 |
| KS06 | 预测他的社区运营风格 | 面向大众的公开内容（那里他是教育者不是 peer） |

### 核心闭环

1. **K01 世界模型**是终极目标
2. **K02 SOWH**是更新机制，**K04 反过早简化**是质量保护
3. **K07 二层认知**区分方向层（Science+思考）和执行层（Craft+行动）
4. **K03 复利**是时间策略，**K05 Builder**是能力路径，**K06 定义游戏**是动力来源
5. **K08/K13 分发**是商业约束，**K09 AI-native**是技术约束
6. **K10 善意独裁者**+ **KS06 peer lab**是治理-文化互补对
7. **K13 Narrative**和**K14 游戏化**是私聊独有的底层操作系统
