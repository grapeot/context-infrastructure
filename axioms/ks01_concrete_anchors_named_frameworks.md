# KS01 具体锚点 + 命名框架

## 核心表述

教学的默认结构是：用具体的数字、故事或场景作为锚点引入话题，然后提炼为一个有编号和命名的框架（"三个核心能力"、"二层模型"、"OPC 框架"）。框架命名和分类学是他组织世界的基本方式——几乎所有内容的终点都是一个结构化的、可引用的框架。

## 展开

课代表的内容有一个可辨识的节奏：先讲一个具体的故事（小张在公司被老板抢功劳、他自己买车时的思考过程、餐馆的成本结构），然后从中提炼出一个有编号的原则框架。故事让框架有血肉，框架让故事有骨骼。

他特别偏好数字先行——复利视频开场不是讲道理而是直接算 1.9×0.7 vs 1.2×1.2；AI 视频引用"90% 的 Claude Code 代码是 Claude Code 写的"、"每天 5 个 PR"。数字比叙述更难被质疑，这与他"反鸡汤反大词"的倾向一致。

框架命名是他最一致的知识组织方式。他几乎不会让一个洞察停留在散文状态——总是收束为一个有名字、有编号的结构。这使得他的内容易于记忆和传播（"OPC 框架"、"三个开关"、"二层模型"比一段论述更容易被引用）。

## 边界条件

- **Taxonomy/lecture 模式是例外**：当视频本身就是一个框架讲解（经济学三条启示、32 条成功要素），他会框架先行，具体例子后置。约 40% 的内容是这种模式。
- **访谈格式**：嘉宾主导的访谈中，故事由嘉宾提供，课代表的角色是从嘉宾故事中提炼框架。
- **时间演化**：早期（2020-2021）更偏教授式框架先行，后期（2023+）更偏故事先行。

## 代表性证据

1. "你1.9乘以0.7是1.33......1.2乘以1.2是1.44" — 用数字开场，再讲复利原理 [YouTube: 复利, 2024-04-13]

2. "90%的Claude Code的代码是Claude Code写的......平均每天每人要merge 5个pull request" — 数据先行 [YouTube: AI怀疑论者失语, 2025-09-25]

3. 世界模型帖子的结构：从可口可乐的认知差异故事 → 提炼"世界模型"概念 → 给出科学方法的操作框架 [Circle: 世界模型, 2024-09-18]

4. "成长的三个开关"、"OPC 框架（产品×销售×效率）"、"二层认知模型" — 命名框架的典型产出 [多源]
